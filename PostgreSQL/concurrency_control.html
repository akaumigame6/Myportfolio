PostgreSQLで挑む同時実行制御のサイバー・ミステリー

【第1章】SFミステリー編：名探偵と「書き換わる密室」

>> SCENARIO:
電脳屋敷のデータセクタ（evidenceテーブル）に、「犯人はヤス」というデジタル・ダイイングメッセージが残されていた。
ネットランナー探偵（A）がアクセスしたその背後で、ステルス怪盗（B）が侵入し、データを書き換えてしまう……。

実験シナリオ：Read Committedの恐怖

デフォルトの隔離レベルでは、接続中（トランザクション中）であっても、現実は歪められてしまう。

> Terminal A (Detective Netrunner) [STATUS: CONNECTED]
BEGIN; SELECT * FROM evidence; -- 結果: "犯人はヤス" -- (ここでBが動く) SELECT * FROM evidence; -- WARNING: データ不整合を検知! "犯人は犯人" に変化!
> Terminal B (Phantom Hacker) [STATUS: INTRUDING]
-- Aの背後で実行 UPDATE evidence SET message = '犯人は犯人' WHERE id = 1; COMMIT; -- 書き換え完了。ログアウト。

これが Non-Repeatable Read（読み取り不整合） 現象だ。
